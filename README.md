# GatewayRAG

A self-hosted **multi-provider LLM gateway** with built-in **Retrieval-Augmented Generation (RAG)**, exposing a unified OpenAI-compatible API across OpenAI, Anthropic, Google, and other model providers — with automatic failover, request-level model selection, and document-grounded chat.

---

## What it does

- 🔀 **Multi-provider routing** — one endpoint, many backends (GPT-5, Claude, Gemini, Grok, Llama, Solar, EXAONE).
- 🛡️ **Automatic failover** — if a model fails, the gateway transparently retries the next one in a configurable chain.
- 📄 **RAG out of the box** — upload a PDF, get a chunked + embedded knowledge base, then ask questions grounded in your document.
- 🔍 **Full transparency** — every response includes which model answered, which fallbacks were attempted, and how many context chunks were retrieved.
- 🔐 **Environment-managed credentials** — API keys never touch source code.

## Architecture

```
   Client (Browser / Mobile / CLI)
              │
              ▼
   ┌────────────────────────┐
   │   FastAPI Gateway      │  ← Pydantic validation, API key auth
   └──────────┬─────────────┘
              ▼
   ┌────────────────────────┐
   │   Router + Fallback    │  ← model selection, automatic retry
   └──────────┬─────────────┘
              ▼
   ┌────────────────────────┐
   │   RAG Middleware       │  ← retrieve top-k chunks from ChromaDB
   └──────────┬─────────────┘
              ▼
   ┌────────────────────────┐
   │   LLM Providers        │  ← OpenAI / Anthropic / Google / others
   └────────────────────────┘
```

## Tech stack

| Layer            | Tool                                            |
|------------------|-------------------------------------------------|
| Web framework    | FastAPI + Uvicorn                               |
| LLM SDK          | `openai` (OpenAI-compatible gateway client)     |
| Vector DB        | ChromaDB (persistent, local)                    |
| PDF parsing      | pypdf                                           |
| Validation       | Pydantic                                        |
| Config           | python-dotenv                                   |

## Endpoints

| Method | Path       | Description                                                |
|--------|------------|------------------------------------------------------------|
| GET    | `/`        | Health check                                               |
| POST   | `/upload`  | Upload a PDF → chunk → embed → store in vector DB          |
| POST   | `/chat`    | Send a message; optionally enable RAG and/or pick a model  |

Interactive docs available at `http://127.0.0.1:8000/docs` (auto-generated by FastAPI).

## Quick start

```bash
# 1. Clone
git clone https://github.com/YOUR_USERNAME/gatewayrag.git
cd gatewayrag

# 2. Set up environment
python -m venv venv
.\venv\Scripts\Activate.ps1    # Windows
# source venv/bin/activate     # macOS / Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure secrets
echo "HAI_API_KEY=your_api_key_here" > .env

# 5. Run
uvicorn main:app --reload
```

Open `http://127.0.0.1:8000/docs` and start sending requests.

## Example: RAG chat

```bash
# Upload a document
curl -X POST http://127.0.0.1:8000/upload \
  -F "file=@my_document.pdf"

# Ask a question grounded in the document
curl -X POST http://127.0.0.1:8000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "What is the refund policy?",
    "use_rag": true
  }'
```

Response:
```json
{
  "reply": "According to your document, the refund policy is...",
  "model_used": "gpt-5-mini",
  "fallbacks_attempted": [],
  "rag_used": true,
  "context_chunks_retrieved": 3
}
```
